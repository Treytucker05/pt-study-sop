# Specification: Repo-Native Agent Board

**Track ID:** agent-board_20260306
**Type:** Docs
**Created:** 2026-03-06
**Status:** Draft

## Summary

Create a lightweight repo-native agent coordination board so multiple agents can communicate ownership, status, blockers, artifacts, and handoffs without relying on Slack or browser state.

## Goal

Add one explicit shared surface that answers:

- who is working on what
- which files they own
- what they are blocked on
- where their artifacts or results live
- what the next agent should do

## Acceptance Criteria

- `docs/root/AGENT_BOARD.md` exists and is clearly positioned as the shared multi-agent handoff surface.
- Root `AGENTS.md` and `docs/root/AGENT_SETUP.md` link to it.
- The board format is simple enough to maintain manually in markdown.
- The board does not replace `docs/root/TUTOR_TODO.md` or `conductor/tracks.md`; it complements them.
