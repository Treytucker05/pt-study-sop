# Specification: Skills Hygiene

**Track ID:** skills-hygiene_20260306
**Type:** Infra
**Created:** 2026-03-06
**Status:** Draft

## Summary

Audit the installed skill ecosystem across shared and tool-local homes, group skills into practical buckets, document the canonical shared-skill architecture, and perform a safe first cleanup pass that avoids deleting actively shared skills blindly.

## Context

The machine has multiple skill surfaces:

- `C:\Users\treyt\.agents\skills` as the shared upstream
- tool homes under `.codex`, `.claude`, `.opencode`, and `.cursor`

Most tool-local entries are junctions into `.agents\skills`, not real duplicate directories. There are also tool-specific exceptions and broken Cursor junctions.

## Acceptance Criteria

- One repo doc explains the shared skill architecture and grouping.
- Skills are grouped into actionable buckets:
  - keep/core
  - optional/specialized
  - tool-specific exceptions
  - broken/bad links
  - likely-unused candidates
- Safe first cleanup is applied without deleting shared upstream skills blindly.
- Broken Cursor junctions are either removed or explicitly documented for follow-up.

## Non-goals

- Blind deletion of low-evidence shared skills from `.agents\skills`.
- Reworking tool-specific packages like `.codex/skills/agent-skills` without a separate audit.
