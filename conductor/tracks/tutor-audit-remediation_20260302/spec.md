## Context

## Problem Description

**P0 — Contract Blockers:**
1. Save-wrap URL embeds query param in path segment (`getSessionSummary(\`${sessionId}?save=true\`)`) producing a 404.
2. Session summary response fields don't match frontend expectations (`duration_minutes` vs `duration_seconds`, `artifacts` vs `artifact_count`, `objectives` vs `objectives_covered`).
3. Chain progress returned as string `"2/6 blocks"` instead of structured object.

**P1 — Artifact Command Parity:**
4. `/table` and `/smap` commands produce 400 because backend rejects those artifact types. Mapping to existing types is trivial.

## Acceptance Criteria
