# RQA-120 - Brain Surface Audit

## Scope

- `dashboard_rebuild/client/src/pages/brain.tsx`
- `dashboard_rebuild/client/src/components/brain/`

## Findings

- ID: RQA-120-001
  Severity: P2
  Type: Command handling dead-end
  Surface: `dashboard_rebuild/client/src/components/brain/UnifiedBrainCanvas.tsx`, `dashboard_rebuild/client/src/components/MindMapView.tsx`, `dashboard_rebuild/client/src/components/brain/ConceptMapStructured.tsx`
  Summary: Undo/Redo controls are exposed in the canvas toolbar, but the graph command pipeline never supports those command types in either canvas mode, so the controls are effectively non-functional.
  Repro: Open a graph mode (`mindmap` or `structured`) and inspect the top toolbar. `Undo`/`Redo` buttons are always disabled and remain inert because canvas status never reports history capability.
  Evidence: `UnifiedBrainCanvas.tsx:211-215` dispatches `issueCommand("undo")` and `issueCommand("redo")`; `graph-canvas-types.ts:8-9` includes `undo`/`redo` as command options; `MindMapView.tsx:610-649` and `ConceptMapStructured.tsx:356-383` command switch handlers do not include `undo`/`redo`; both components force `canUndo: false` and `canRedo: false` (`MindMapView.tsx:595-597` and `ConceptMapStructured.tsx:341-343`).
  Likely owner: Frontend (`dashboard_rebuild/client/src/components/brain/`)
  Fix wave: Wave 2

- ID: RQA-120-002
  Severity: P2
  Type: Command contract mismatch
  Surface: `dashboard_rebuild/client/src/components/brain/ConceptMapStructured.tsx`
  Summary: Internal structured toolbox actions use a `delete` action id, while the cross-canvas external command contract uses `delete_selected`; this creates a mismatched command surface and brittle coupling between callers and handlers.
  Repro: Compare invocation paths: `handleToolAction` in `ConceptMapStructured` handles `delete`, but its external command handler only handles `delete_selected`.
  Evidence: `ConceptMapStructured.tsx:656-660` maps toolbox id `delete` to `deleteSelected`; `ConceptMapStructured.tsx:381-383` handles only `delete_selected`; command type is normalized as `delete_selected` in the contract (`graph-canvas-types.ts:11`).
  Likely owner: Frontend (`dashboard_rebuild/client/src/components/brain/ConceptMapStructured.tsx`)
  Fix wave: Wave 3

- ID: RQA-120-003
  Severity: P3
  Type: Node editing UX
  Surface: `dashboard_rebuild/client/src/components/brain/MindMapNodes.tsx`
  Summary: Mind map node labels cannot be intentionally cleared because commit logic strips whitespace and blocks empty labels.
  Repro: Double-click a node to edit, delete all text, blur/focus out.
  Evidence: `MindMapNodes.tsx:90-93` only dispatches `mindmap:node-label` when `editValue.trim()` is truthy and `editValue !== label`, so blank labels are dropped.
  Likely owner: Frontend (`dashboard_rebuild/client/src/components/brain/MindMapNodes.tsx`)
  Fix wave: Wave 3
