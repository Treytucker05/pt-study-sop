# RQA-110 - Shared Shell And Navigation Audit

## Scope

- `dashboard_rebuild/client/src/App.tsx`
- `dashboard_rebuild/client/src/components/layout.tsx`
- shared shell and nav behavior

## Findings

- ID: RQA-110-001
- Severity: P2
- Type: ui
- Surface: Shared router + shell nav active-state mapping (`/`, `/brain`)
- Summary: Router and shell compute active route from different `/brain` matching rules. `App.tsx` only normalizes exact `/brain` to `/`, while `layout.tsx` treats any path starting with `/brain` as `/`. As a result, deep/prefixed paths can render `NotFound` but still keep BRAIN as the active nav item.
- Repro: Open `http://127.0.0.1:5000/brainstorm` (or `/brain/test`), then observe the route content and active shell tab. `NotFound` appears while the BRAIN nav item remains highlighted.
- Evidence:
  - `dashboard_rebuild/client/src/App.tsx:31-35` normalizes only exact `"/brain"`.
  - `dashboard_rebuild/client/src/App.tsx:63-84` treats only exact normalized routes as known and renders `NotFound` when unknown.
  - `dashboard_rebuild/client/src/components/layout.tsx:251-254` normalizes any `location.startsWith("/brain")` to `"/"`.
  - `dashboard_rebuild/client/src/components/layout.tsx:914-952` derive active nav state from `currentPath` without validating route existence.
- Likely owner: frontend
- Fix wave: Wave 2

- ID: RQA-110-002
- Severity: P2
- Type: ux
- Surface: Shared route shell scroll state (`main` + keep-alive router)
- Summary: Navigation uses keep-alive route rendering that preserves mounted components and relies on `display: none` to hide inactive routes, while the shared shell `main` scroll container is persistent. There is no explicit route-change scroll reset, so navigating between routes can retain prior scroll offset and create broken context continuity.
- Repro: With two long routes, scroll midway on the first route, use nav to switch to the second route, and confirm the second route does not reliably start at the top.
- Evidence:
  - `dashboard_rebuild/client/src/App.tsx:67-75` keeps previously visited routes mounted and hides inactive ones with `display: none`.
  - `dashboard_rebuild/client/src/App.tsx:69-70` only toggles visibility via `isActive`; it does not trigger scroll restoration.
  - `dashboard_rebuild/client/src/components/layout.tsx:1318-1322` uses one persistent `main` with `overflow-y-auto` for all routes.
  - `dashboard_rebuild/client/src/components/layout.tsx:251-257` derive `currentPath` but never reset scroll position on route changes.
- Likely owner: frontend
- Fix wave: Wave 2
