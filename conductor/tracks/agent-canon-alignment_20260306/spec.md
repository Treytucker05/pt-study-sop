# Specification: Agent Canon Alignment and Subagent Reliability

**Track ID:** agent-canon-alignment_20260306
**Type:** Infra
**Created:** 2026-03-06

## Summary

Repair the broken Codex subagent configuration, establish one explicit project instruction canon, align repo-local and home-directory agent entrypoints to inherit it consistently, and document the final setup so future agent changes do not drift.

## Context

The PT Study repo already treats root `AGENTS.md` as the canonical project instructions, but the active agent surfaces are split across:

- repo `AGENTS.md` / `CLAUDE.md`
- repo `.claude/AGENTS.md`, `.claude/CLAUDE.md`, `.claude/agents/*`, `.claude/commands/*`
- global `C:\Users\treyt\.claude\CLAUDE.md`, `.claude\rules\*`, `.claude\agents\*`
- global `C:\Users\treyt\.codex\AGENTS.md`, `.codex\config.toml`, `.codex\agents\*.toml`, `.codex\rules\default.rules`

Current mismatch:

1. Codex subagent roles (`awaiter`, `explorer`, `worker`) use `gpt-5.3-codex-spark`.
2. Global Codex config enables `model_reasoning_summary = "concise"`.
3. Spawned subagents fail because that model path rejects `reasoning.summary`.

## Problem Description

1. The current Codex subagent config is broken for real worker execution.
2. Repo and global agent files do not all explicitly inherit the same canon.
3. Some surfaces are compatibility shims while others carry role logic, but that split is not documented or enforced.
4. The repo lacks one operator-facing document that explains exact precedence, ownership, and safe extension points.

## Acceptance Criteria

- Codex subagents can spawn without the `reasoning.summary` error.
- Root `AGENTS.md` remains the project canon and every active repo/global agent surface either defers to it or is explicitly documented as global-only.
- Repo `.claude/agents/*` and global `.claude/agents/*` follow one inheritance template.
- Repo command files reference valid agent IDs after normalization.
- The final setup is documented in `docs/root/AGENT_SETUP.md`.
- Drift validation and cross-tool smoke checks pass after the cleanup.

## Non-goals

- Renaming the repo or global agent catalogs in this pass.
- Broad model tuning unrelated to the broken Codex subagent path.
- Refactoring archived worktree copies under `.claude/worktrees/`.
