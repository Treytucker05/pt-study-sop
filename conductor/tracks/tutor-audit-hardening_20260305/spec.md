# Specification: Tutor Audit Hardening

**Track ID:** tutor-audit-hardening_20260305
**Type:** Bug
**Created:** 2026-03-05
**Status:** Draft

## Summary

Implement the validated Tutor audit remediation and associated medium-priority UI/coverage gaps so tutor session behavior, context/debug telemetry, and artifact handoff remain correct under reuse and recovery scenarios.

## Context

The latest tutor audit identified high-priority defects that can lose data correctness or break UX behavior, plus related medium-priority gaps that block safe operations and regressions detection.

## Problem Description

### High-priority defects

1. Tutor session deletion can delete shared learning objectives because cleanup uses a coarse `course_id + lo_code` match.
2. Instruction-context retrieval wiring is effectively dead in runtime while `api_tutor` and prompt code still imply instruction-specific semantics.
3. Retrieval telemetry/debug is misleading due to key mismatch (`materials_debug` vs `materials`) and empty stub doc lists.
4. Library-to-Tutor handoff uses split localStorage keys (`v1` / `v2`) and can lose selected materials across navigation.

### Medium-priority gaps

5. Resume restore drops valid artifact types like `structured_notes`.
6. Delete telemetry is not surfaced in Tutor UI beyond generic success/error.
7. Malformed SSE chunks are silently swallowed, obscuring stream regressions.
8. Dead `otherMaterials` branch remains in `MaterialSelector`.
9. Missing regression coverage for bulk delete, session-wrap, and handoff recovery.

## Acceptance Criteria

- Deleting one tutor session never removes learning objectives still referenced by another tutor session or manual/dashboard workflow.
- Tutor runtime/debug output reflects the actual retrieval path and matches the built shape.
- Library-selected materials survive navigation into Tutor.
- Restored sessions preserve backend-valid artifact types used by Tutor.
- Partial delete/reporting UX shows request correlation and actionable counts/reasons.
- Regression coverage exists for all repaired flows, including malformed SSE and storage handoff.

## Non-goals

- Reintroducing a second instruction vector collection or restoring separate instruction-specific retrieval plumbing.
