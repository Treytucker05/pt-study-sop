What flows where

READ
Source Shelf → Document Dock (open a doc) → Workspace (drag in excerpts)
THINK
Workspace + Notes arrange excerpts, sketches, repair notes → Prime Packet
RUN
Prime Packet (inputs) + Run Config → Priming sets up → Tutor runs the live session
CAPTURE
Tutor output → Polish Packet (outputs) → Polish reviews + packages
EXPORT
Polish Packet → Notes, Obsidian (vault), Anki (cards)
CROSS-CUTTING
Run Config shapes Priming + Tutor. Memory resumes the next session.

BEFORE — flat strip, two collisions

  • 13 panel chips in one row, no grouping — eye has nothing to anchor on
  • "Priming" and "Polish" appear in both rows (panel vs preset) — same word, different verb
  • Selection chip shows "0 Selected" forever — noise
  • Group / Ungroup stranded between panels and presets
  • 5 preset chips compete with 13 panel chips for visual weight
  • Prime Packet vs Polish Packet — easy to confuse by name alone

AFTER — pipeline reads left-to-right

  • 5 named zones: READ → THINK → RUN → CAPTURE → EXPORT
  • Faint → arrows between zones teach order in one glance
  • Layout ▾ dropdown ends the Priming/Polish name collision
  • Selection chip only appears when N ≥ 1
  • Group / Select / Arrange on row 2 — clear scope progression
  • Run Config + Memory pulled to a cross-cutting cluster — they're not pipeline stages
  • Stage label glows when a panel from that stage is open — toolbar = progress indicator
