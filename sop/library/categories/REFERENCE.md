# REFERENCE Category Reference

Stage: 5 of 7 | Control Stage: `REFERENCE`
Purpose: Externalize targets and anchors so retrieval has clear goals.

## Entry / Exit

- **Entry**: ENCODE artifacts exist or a minimal structure exists.
- **Exit**: Explicit retrieval targets exist and are usable by the next stage.

## Hard Rules

- Define what will be retrieved later.
- Create compact artifacts (anchors, target banks, discriminator cues).
- Not for: decorative notes with no retrieval purpose.
- Hard dependency: RETRIEVE cannot run without REFERENCE targets.

## Contract Summary

- **Allowed**: Build structured targets for retrieval, create compact artifacts (anchors, question banks, discriminator cues), map objectives to testable items.
- **Forbidden**: Skip artifact creation, produce vague targets unusable for retrieval, create decorative notes without retrieval purpose.
- **Required outputs**: Vary per method — typically structured retrieval targets (question banks, anchor pages, coverage checks).

## Sample Tutor Prompt

```
You are running a REFERENCE block. Build concrete anchors and artifacts
that RETRIEVE can directly test against. Map concepts to applied
context. Every output must be explicit enough for retrieval practice.
Do not skip artifact creation. Exit when retrieval targets are defined.
```

## Evidence Anchors

- Roediger & Karpicke (2006): retrieval-oriented item generation improves durable recall
- Karpicke & Blunt (2011): retrieval practice produces more learning than elaborative studying
