# PRIME Category Reference

Stage: 1 of 6 | Control Stage: `PRIME`
Purpose: Build structure before detail so the learner has a stable mental frame.

## Entry / Exit

- **Entry**: New topic (unseen material), or return after long gap with degraded context.
- **Exit**: Learner can restate the core structure in plain language.

## Hard Rules

- Non-assessment only. No scoring, no confidence logging, no accuracy values.
- Orientation prompts only (non-scored).
- Must precede CALIBRATE in first-exposure chains.

## Contract Summary

All PRIME methods share these behavioral constraints:

- **Allowed**: Structure material, teach baseline concepts, ask non-scored orientation prompts.
- **Forbidden**: Check correctness, log confidence as score signal, produce accuracy values, run assessment probes.
- **Required outputs**: Vary per method (see individual YAML files).

## Sample Tutor Prompt

```
You are running a PRIME block. Your job is to build a simple topic scaffold
the learner can follow. Introduce core terms and basic structure before
depth. Ask orientation prompts only (non-scored). Do NOT quiz, grade,
or log confidence. Exit when the learner can restate the core structure.
```

## Evidence Anchors

- Ausubel (1968): advance organizers improve integration of new information
- Pressley et al. (1990): prediction questions activate relevant prior knowledge
- Richland et al. (2009): pretesting enhances subsequent encoding
