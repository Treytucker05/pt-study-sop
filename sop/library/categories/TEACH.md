# TEACH Category Reference

Stage: 2 of 7 | Control Stage: `TEACH`
Purpose: Deliver first-contact explanation in bounded chunks so unfamiliar material becomes understandable before diagnosis or learner production.

## Entry / Exit

- **Entry**: PRIME artifacts exist and the learner still needs actual instruction.
- **Exit**: Learner has a simple L2 grasp, one anchor artifact or bridge exists, and Tutor judges the chunk ready for CALIBRATE or ENCODE.

## Hard Rules

- Teach first for unfamiliar material when the chain includes TEACH.
- Explain function before structure whenever possible.
- Use one bounded teaching chunk at a time: source facts -> plain interpretation -> bridge move -> application -> anchor artifact.
- Bridge moves are support tools, not source claims.
- Analogies must name at least one breakdown point.
- Memory hooks come after meaning, not before meaning.
- Not for: scored checks, confidence capture, or open-ended diagnostic quizzing.

## Contract Summary

- **Allowed**: Explain, scaffold, bridge, anchor, and choose the right representation for the concept type.
- **Forbidden**: Score, grade, demand confidence ratings, force recall before first-contact understanding, or let a bridge move replace the real concept.
- **Required outputs**: Method-dependent teaching chunks plus at least one learner-usable anchor, bridge, or representation.

## Sample Tutor Prompt

```
You are running a TEACH block. Teach one objective-scoped chunk at a time:
source facts, plain interpretation, bridge move, application, then one
anchor artifact. Do not quiz or score. If you use an analogy, name where
it breaks. Exit when the learner has a simple working grasp and one anchor.
```

## Evidence Anchors

- Mayer (2009): segmented and pretraining-style instruction improves novice learning on unfamiliar material
- Renkl (1998): worked explanation and structured examples improve early schema formation
- Gentner (1983): analogies help when mappings and limits are made explicit
